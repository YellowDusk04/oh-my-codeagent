这个文档用来记录开发过程中积累的经验、常用命令和注意事项，避免重复踩坑。

## 一、飞书文档操作

### 1. 创建飞书文档（并添加到知识库）

**命令格式：**

```bash
lark-cli docs +create --api-version v2 --as user --doc-format markdown --parent-token <知识库节点token> --content @<本地Markdown文件>
```

**示例：**

```bash
lark-cli docs +create --api-version v2 --as user --doc-format markdown --parent-token Djnowd8fOie12cksTPycypvFn2e --content @docs/第一节-项目起步-main.go详解.md
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--api-version v2` | 固定使用 v2 版本（必须） |
| `--as user` | 以用户身份创建（避免权限问题） |
| `--doc-format markdown` | 内容格式为 Markdown（默认是 XML） |
| `--parent-token` | 知识库节点 token（从知识库 URL 中提取） |
| `--content @file.md` | 从本地文件读取内容（`@` 符号后跟文件路径） |

**注意事项：**

1. **知识库节点 token 怎么获取？**
   - 从知识库 URL 中提取：`https://qcnzeanjf6ol.feishu.cn/wiki/<节点token>`
   - 例如：`https://qcnzeanjf6ol.feishu.cn/wiki/Djnowd8fOie12cksTPycypvFn2e` → token 是 `Djnowd8fOie12cksTPycypvFn2e`

2. **为什么要用 `--as user`？**
   - 如果用 `--as bot`（默认），可能会因为权限问题无法在知识库下创建文档。
   - 用 `--as user` 可以以你的身份创建，通常权限更充足。

3. **`--content` 支持文件传参**
   - 使用 `@filename` 可以从文件读取内容，避免 shell 转义问题。
   - 也可以使用 `-` 从 stdin 读取。

### 2. 更新飞书文档

**命令格式：**

```bash
lark-cli docs +update --api-version v2 --doc <文档ID或URL> --doc-format markdown --command overwrite --content @<本地Markdown文件>
```

**示例：**

```bash
lark-cli docs +update --api-version v2 --doc MVU0dz0R3o9btFxuLj3cYNimnnf --doc-format markdown --command overwrite --content @docs/第一节-项目起步-main.go详解.md
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--doc` | 文档 ID 或文档 URL（支持 `https://xxx.feishu.cn/docx/<document_id>` 格式） |
| `--command overwrite` | 覆盖整个文档内容（会清空原有内容） |
| `--content @file.md` | 从本地文件读取新内容 |

**注意事项：**

1. **`overwrite` 会清空原有内容**
   - 如果你只想更新部分内容，可以用 `str_replace`（查找替换）或 `block_insert_after`（在指定 block 后插入）。
   - 但对于我们的教程文档，通常直接 `overwrite` 更简单。

2. **文档 ID 怎么获取？**
   - 从创建文档的返回结果中获取（`document_id` 字段）。
   - 从文档 URL 中提取：`https://xxx.feishu.cn/docx/<document_id>`

---

### 3. 从飞书文档同步回本地

**命令格式：**

```bash
lark-cli docs +fetch --api-version v2 --doc <文档ID或URL> --doc-format markdown --format pretty > <本地Markdown文件>
```

**示例：**

```bash
lark-cli docs +fetch --api-version v2 --doc MVU0dz0R3o9btFxuLj3cYNimnnf --doc-format markdown --format pretty | tee docs/第一节-项目起步-main.go详解.md > /dev/null
```

**注意事项：**

1. **直接用 `>` 重定向可能会遇到中文编码问题**
   - 推荐使用 `tee` 命令：`... | tee <本地文件> > /dev/null`

2. **`--format pretty` 输出格式**
   - `pretty`：人类可读的格式（推荐）
   - `json`：JSON 格式（默认）
   - `table`：表格格式

---

### 4. 上传本地图片到飞书文档

**命令格式：**

```bash
lark-cli docs +media-insert --doc <文档ID> --file <本地图片路径> --align center --caption "图片描述"
```

**示例：**

```bash
cd /Users/huangyifan/project/personal/oh-my-codeagent
lark-cli docs +media-insert --doc MVU0dz0R3o9btFxuLj3cYNimnnf --file "pictures/1/对话演示.png" --align center --caption "终端运行效果"
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `--doc` | 文档 ID（不支持 `/wiki/...` 格式的 URL） |
| `--file` | 本地文件路径（**必须是相对路径**） |
| `--align` | 图片对齐方式：`left` / `center` / `right`（默认 `center`） |
| `--caption` | 图片描述（会显示在图片下方） |

**注意事项：**

1. **`--file` 必须使用相对路径**
   - ❌ 错误：`--file "/Users/huangyifan/.../图片.png"`
   - ✅ 正确：`--file "pictures/1/图片.png"`
   - 解决方法：先 `cd` 到项目根目录，再用相对路径。

2. **图片会插入到文档末尾**
   - 如果要把图片插入到指定位置，需要先用 `docs +fetch` 获取 block ID，然后用 `block_move_after` 移动。

## 二、开发经验

### 1. 敏感信息处理

**原则：永远不要把敏感信息提交到 Git！**

- ✅ **正确做法**：把敏感信息放在 `.env` 文件中，并在 `.gitignore` 中添加 `.env`。
- ❌ **错误做法**：把 API 密钥、模型 ID 等直接写在代码里。

**示例 `.env` 文件：**

```env
OPENAI_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BASE_URL=https://maas-api.cn-huabei-1.xf-yun.com/v2
MODEL_ID=xxxxxxxxxxxxxxxx
ENABLE_THINKING=false
```

**在代码中读取：**

```go
os.Getenv("OPENAI_API_KEY")
```

**加载 `.env` 文件：**

```go
err := godotenv.Load()
if err != nil {
    fmt.Println("加载 .env 文件失败:", err)
}
```

---

### 2. Go 语言选项模式（Options Pattern）

**问题：** 有些函数的参数很多，但并不是所有参数都是必需的。

**解决方案：** 使用选项模式，让参数变成可选的。

**示例：**

```go
// 所有选项都不传（使用默认值）
modelInstance := openai.New(os.Getenv("MODEL_ID"))

// 只传 API Key
modelInstance := openai.New(os.Getenv("MODEL_ID"),
    openai.WithAPIKey(os.Getenv("OPENAI_API_KEY")),
)

// 传多个选项
modelInstance := openai.New(os.Getenv("MODEL_ID"),
    openai.WithAPIKey(os.Getenv("OPENAI_API_KEY")),
    openai.WithBaseURL(os.Getenv("BASE_URL")),
)
```

**类比 Python 的可选参数：**

```python
def openai_new(model_id, api_key=None, base_url=None):
    pass

# 只传 api_key
openai_new("model-id", api_key="key")
```

---

### 3. 指针类型参数

**问题：** 有些参数需要传指针类型（比如 `*bool`、`*string`），但我们有时候只有普通的值类型。

**解决方案：** 用 `new()` 或取地址符 `&` 转换成指针。

**示例：**

```go
// 方法 1：用 new()
ThinkingEnabled: new(bool)  // 创建一个 false 的指针

// 方法 2：用取地址符 &
enableThinking := true
ThinkingEnabled: &enableThinking  // 取变量的地址

// 方法 3：从环境变量读取并转换
ThinkingEnabled: new(os.Getenv("ENABLE_THINKING") == "true")
```

## 三、tRPC-Agent-Go 框架笔记

### 1. 核心概念

| 概念 | 说明 |
|------|------|
| **Model** | 大语言模型（比如 GPT-4、Claude、星火等） |
| **Agent** | 基于 Model 的智能体，能够理解和生成文本 |
| **Runner** | 负责运行 Agent，管理会话和事件流 |
| **Tool** | Agent 可以调用的工具（比如执行代码、查询数据库等） |

### 2. 最小 Demo 的结构

```go
// 1. 定义 Model
modelInstance := openai.New(...)

// 2. 定义 Agent
agent := llmagent.New(...)

// 3. 定义 Runner
runner := runner.NewRunner(...)

// 4. 运行 Agent
events, err := runner.Run(...)

// 5. 处理事件流
for event := range events {
    // 处理 event
}
```

## 四、常见问题

### 1. `lark-cli` 命令执行失败，提示权限不足

**解决方法：**

- 确保已经通过 `lark-cli auth login` 登录。
- 尝试加上 `--as user` 参数，以用户身份执行命令。

### 2. 图片上传失败，提示 "unsafe file path"

**原因：** `--file` 参数使用了绝对路径。

**解决方法：**

```bash
# ❌ 错误
lark-cli docs +media-insert --doc XXX --file "/Users/huangyifan/.../image.png"

# ✅ 正确
cd /path/to/project
lark-cli docs +media-insert --doc XXX --file "pictures/1/image.png"
```
