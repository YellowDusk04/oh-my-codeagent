## 为什么需要链路追踪？

第一节的 Agent 是个黑盒：模型调了几次？每次花了多久？输入输出各多少 token？一概不知。

链路追踪就是把这个黑盒打开，把每次调用的完整链路记录下来。

我们直接用 **Langfuse**（https://cloud.langfuse.com），注册就能用，专为 LLM 场景设计，能看到 token 数、成本这些字段。

---

## 配置环境变量

在 `.env` 里加上 Langfuse 的配置：

```env
# 新增这三行
LANGFUSE_PUBLIC_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
LANGFUSE_SECRET_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
LANGFUSE_HOST=xxxxxx
```

> Key 在 Langfuse 控制台的 Settings → API Keys 里创建。

## 改动 main.go：启动 Langfuse

核心就两步：启动遥测，往 Context 里塞 Baggage。

```go
func main() {
	err := godotenv.Load()
	if err != nil {
		log.Println("加载 .env 文件失败:", err)
	}

	// 启动 Langfuse 遥测
	clean, err := langfuse.Start(context.Background())
	if err != nil {
		log.Printf("Failed to start Langfuse telemetry: %v", err)
	} else {
		defer func() {
			if err := clean(context.Background()); err != nil {
				log.Printf("Failed to clean up: %v", err)
			}
		}()
	}

	// ... 创建 model、agent、runner ...

	// 用 Baggage Context 替代 context.Background()
	userID := "user-001"
	sessionID := fmt.Sprintf("session-%d", time.Now().Unix())
	ctx := NewBaggageContext(userID, sessionID)

	events, err := runner.Run(ctx, userID, sessionID, model.NewUserMessage("你是谁呀?"))
	// ...
}
```

`langfuse.Start()` 会读取环境变量，在后台把追踪数据发出去。`defer clean(...)` 确保程序退出前把缓冲区刷掉，避免丢数据。

## 新建 tracing.go：封装 Baggage

**Baggage 是什么？** 简单说就是"随请求链路自动传播的数据"。你可以把它理解成给每次请求贴标签，这些标签会沿着整个调用链一路传下去，不用你手动透传。

Langfuse 靠两个标签来归因：

| Key | 含义 |
|-----|------|
| `langfuse.user.id` | 用户 ID |
| `langfuse.session.id` | 会话 ID |

```go
package main

import (
	"context"
	"log"

	"go.opentelemetry.io/otel/baggage"
)

func NewBaggageContext(userID, sessionID string) context.Context {
	mSession, err := baggage.NewMemberRaw("langfuse.session.id", sessionID)
	if err != nil {
		log.Printf("Failed to create baggage member for session: %v", err)
	}
	mUser, err := baggage.NewMemberRaw("langfuse.user.id", userID)
	if err != nil {
		log.Printf("Failed to create baggage member for user: %v", err)
	}

	bag, err := baggage.New(mSession, mUser)
	if err != nil {
		log.Printf("Failed to create baggage: %v", err)
	}

	return baggage.ContextWithBaggage(context.Background(), bag)
}
```

三步：创建成员 → 打包成 Baggage → 绑到 Context 上。之后所有从这个 Context 派生的操作（模型调用、工具调用）都会携带这些标签。
